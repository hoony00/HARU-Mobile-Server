#### _브랜치_ <p>

- 기능 개발 : develop <p>

- 어떻게 보다는 무엇과 왜를 설명한다 <p>

#### _커밋_ 유형 지정 <p>
- FEAT : 새로운 기능의 추가 <p>
- FIX : 버그 수정 <p>
- DOCS : 문서 수정 <p>
- STYLE : 스타일 관련(코드 포맷팅, 코드 누락, 코드 자체의 변경이 없는 경우 <p>
- REFACTOR : 코드 리팩토링 <p>
- TEST : 테스트 코드, 리팩토링 테스트 코드 <p>
- CHORE : 빌드 업무 수정, 패키지 매니저 수정 <p>
